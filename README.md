# Lanty AI

A fine-tuned LLM that plays **Lanty**, a sentient mushroom NPC who runs The Last Light, an inn at the edge of the Wilds in the Flamebound world of Lithilian.

Built on **Qwen2.5-7B-Instruct** with a LoRA adapter trained on hand-written gold dialogues plus ~5,500 synthetic dialogues generated by Claude. Quantized to Q4_K_M GGUF for local inference via llama.cpp.

The Flamebound setting and canonical lore live in the [Forge Link repo](https://github.com/Jca56/forge_link_app).

## Pipeline

```
data/lore + data/lanty_voice          ┐
                                      │  scripts/generate_training_data.py
                                      │  (Claude API, prompt-cached)
                                      ▼
                         data/training/batch_*.txt  (XML <player>/<lanty>)
                                      │
                                      │  scripts/prepare_chat_dataset.py
                                      ▼
                         data/lanty_chat.jsonl  (HF chat format)
                                      │
                                      │  scripts/train_lanty.py  (LoRA on Lambda)
                                      ▼
                         models/lanty-qwen-lora/
                                      │
                                      │  scripts/merge_and_export.py + llama.cpp
                                      ▼
                         models/lanty-qwen-Q4_K_M.gguf  (1.9 GB)
                                      │
                                      │  inference/chat.py
                                      ▼
                              local REPL with Lanty
```

## Quick start

### 1. Generate synthetic training data

```bash
cd scripts && python -m venv .venv && source .venv/bin/activate
pip install anthropic
export ANTHROPIC_API_KEY=sk-ant-...
python scripts/generate_training_data.py --batches 100 --per-batch 15
```

Existing batches in `data/training/` are skipped on resume; gaps in numbering are filled before adding new batches. Pass `--no-resume` to ignore existing files.

### 2. Prepare the dataset

```bash
python scripts/prepare_chat_dataset.py
```

Combines hand-written gold (`data/lanty_voice/dialogues_*.txt`) + synthetic batches into `data/lanty_chat.jsonl`. Holds out a stratified 5% per mode as `data/lanty_chat_eval.jsonl`. Gold dialogues always go to train.

### 3. Train + export on Lambda Cloud

```bash
./lambda/deploy_python.sh <INSTANCE_IP>
```

8 steps: SSH wait → GPU check → upload → install ML stack → prepare dataset → LoRA fine-tune → merge + GGUF + quantize → download model.

Single-GPU H100 takes ~30–45 min for 5 epochs over ~4k conversations.

### 4. Chat locally

```bash
python -m venv inference/.venv
source inference/.venv/bin/activate
pip install llama-cpp-python   # add CMAKE_ARGS="-DGGML_CUDA=on" for GPU
python inference/chat.py
```

Commands inside the REPL: `/quit`, `/reset`, `/history`. Ctrl-C mid-stream interrupts and saves the partial response.

## Dialogue modes

Each synthetic batch is generated in one of seven modes:

| Mode | Description |
|------|-------------|
| `goofy` | Default — quirky, playful, unhelpful advice |
| `lockedin` | Triggered by "for real" / "seriously" — accurate Flamebound lore |
| `inn_events` | Reactions to tavern situations and guests |
| `emotional` | Lanty's emotional range — supportive moments |
| `game_events` | Reactions to in-game mechanics (combat, level-up, etc.) |
| `glitch` | Lanty accidentally knows Linux/Arch/programming |
| `multi_turn` | 3–5 exchange conversations spanning multiple modes |

Mode weights are set in `scripts/generate_training_data.py:MODE_WEIGHTS`.
