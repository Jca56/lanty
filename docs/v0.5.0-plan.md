# v0.5.0 Plan

Working notes for the next release. Refined as we go.

## Carryover from v0.4.0

- **Bash → Python deploy rewrite** (deferred from v0.4.0 plan)
  - `lambda/deploy_python.sh` is 225 lines of bash with embedded Python heredocs.
  - Goal: replace with a `lambda/deploy.py` orchestrator that drives the same 8 steps via `subprocess` + a thin SSH helper. Easier to test, easier to extend, no shell-escape footguns.
  - Behavior must stay identical: SSH wait → GPU check → upload → install → prepare → train → merge+GGUF → download.

## Driven by v0.4.0 training results

- **Default epochs: 5 → 2.**
  - The v0.4.0 eval curve (see `memory: project_training_baseline`) found the best checkpoint at step 400 / epoch 1.68. Training past epoch ~2 actively makes the model worse.
  - Update `lambda/deploy_python.sh` (or `deploy.py`) `EPOCHS` default and `train_lanty.py` `--epochs` default to 2.
  - Drop `--save-steps` from 100 to 50 for finer eval granularity around the sweet spot.

- **Length distribution fix.**
  - Current model trends long despite training data targeting 20/25/35/20 (very-short / short / medium / long). Runtime length nudge in `chat.py` is a band-aid.
  - Investigate training data: does the actual generated distribution match the prompt targets? If not, rebalance `prompts.SINGLE_TURN_RULES` or post-filter generated batches.
  - Keep the runtime nudge as a fallback either way.

## New surface area

- **Hugging Face Hub integration.**
  - Push GGUF + LoRA adapter to `Jca56/lanty` (private or public — TBD).
  - Add `scripts/setup_local.sh`: creates `inference/.venv`, installs `llama-cpp-python` (CPU default, `--cuda` flag), downloads the model from HF.
  - Goal: `git clone && ./scripts/setup_local.sh && lanty` works on a fresh machine.

- **Run the eval harnesses for real.**
  - First execution of `eval_voice.py` (50/mode Claude judge, $10 cap) against the v0.4.0 model. Commit the JSON output to `data/eval/` as a baseline.
  - First execution of `eval_glitch.py` after generating + auto-reviewing probes. Commit the probe cache + result JSON.
  - Future training rounds compare against the v0.4.0 baseline.

- **More gold dialogues.**
  - Hand-written, no Claude middleman. These go 100% to train and shape the voice the most.
  - Target: +50 conversations across underrepresented modes (especially `emotional`, `glitch` direct-asks).

## Open questions

- [ ] **Gitignore the synthetic batches?** `data/training/batch_*.txt` is 286 files. They're regenerable from the seed + Claude API. Tracking them bloats the repo (~3 MB now, will grow). Decision: keep tracked for reproducibility, or move to `data/training/.gitignore` and treat as build artifacts?
- [ ] **HF repo public or private?**
- [ ] **Continued pretraining on lore corpus?** Run a small pretraining pass on `lore_v2.md` + species/creatures JSON before the SFT stage to push the base toward "Lithilianspace". Advanced — scope creep risk.
- [ ] **More glitch domains?** Lanty currently glitches into Linux/programming. Add cooking chemistry, woodworking, herbalism, etc.? Each new domain needs its own probe set in `eval_glitch.py`.
- [ ] **Eval harnesses in CI?** Probably overkill for a solo project, but worth flagging.

## Out of scope for v0.5.0

- Training from scratch (cost/time prohibitive, would produce a worse model).
- Switching base model away from Qwen2.5-7B (no clear reason yet).
- Multi-character training (Lanty is the one for now).
